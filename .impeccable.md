# Shadpedia Design Context

## Design Context

### Users
Developers searching for shadcn/ui components across multiple registries (shadcn/ui, Magic UI, Aceternity UI, etc.). Technical audience, comfortable with terminals and CLIs. `/spadmin` is internal-only, used by the creator.

### Brand Personality
**Three words:** Archival. Structural. Precise.

Emotional target: **nostalgic** — the calm authority of a Wikipedia infobox and the tactile quality of a 1994 tech manual, rendered with modern CSS. Not retro kitsch.

### Aesthetic Direction

- **Reference implementation**: [apps/web/src/app/page.tsx](apps/web/src/app/page.tsx) is the canonical system. New pages must match its patterns (border-2 cards, hard-shadow hover, uppercase-tracking-widest labels, Wikipedia-style infobox `<dl>` for stats).
- **Signature moves**: `border-2 border-foreground` as primary structure, `hard-shadow` (4px hard black offset) as the only hover/focus effect, Libre Bodoni serif headings on Public Sans body, zero border radius.
- **Theme**: Light-first (paper). Dark mode is supported but light is the intended primary experience.
- **Anti-references**: Every other shadcn site, Vercel/Linear minimalism, SaaS hero gradients, retro kitsch (scanlines, terminal chrome, ASCII borders), persistent-sidebar doc layouts.

### Design Principles

1. **Borders are load-bearing.** `border-2 border-foreground` is the structural voice; use it generously on surfaces.
2. **Uppercase + tracking-widest is our small-caps.** Stay inside that label convention.
3. **One hover effect.** `hard-shadow` appearing on hover/focus is the signature; no scale, lift, or additional shadows.
4. **Infobox over metric grid.** Stats go in a `<dl>` inside a single infobox, never a 3-column stat-card row.
5. **Hero centered; everything else left-aligned.** Center alignment is a rare tool.
6. **Accent color is rare.** `text-accent` for source labels and active states only; `text-primary` for prose links only.
